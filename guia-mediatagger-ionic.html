Introducción a MediaTagger

MediaTagger es una aplicación móvil multiplataforma que permite a los usuarios capturar fotos, organizarlas mediante etiquetas personalizadas y gestionar una galería inteligente de imágenes.

💡 Características Principales
  • 📸 Captura de fotos usando la cámara del dispositivo
  • 🏷️ Sistema de etiquetado flexible y personalizable
  • 🖼️ Galería visual con filtrado por etiquetas
  • 📱 Interfaz responsive para móvil y web
  • 💾 Almacenamiento local de imágenes y metadatos
  • 🔍 Búsqueda y filtrado avanzado

🎯 Objetivos del Proyecto

Bases Teóricas de Ionic

¿Qué es Ionic Framework?

Ionic es un SDK (Software Development Kit) de código abierto para el desarrollo de aplicaciones híbridas multiplataforma. Su filosofía principal es "Escribe una vez, corre en todas partes" (Write Once, Run Anywhere). Permite a los desarrolladores web usar tecnologías que ya conocen (HTML, CSS y JavaScript/TypeScript) para crear aplicaciones de alta calidad para iOS, Android, Web (incluyendo PWAs) y escritorio, todo desde una única base de código.

A diferencia de soluciones nativas (como Swift/Kotlin) o soluciones "cross-compiled" (como React Native), Ionic utiliza un enfoque basado en Web Standards. La aplicación se ejecuta dentro de un contenedor nativo (como Capacitor o el antiguo Cordova), que actúa como un puente para acceder a las funciones nativas del dispositivo (cámara, GPS, sistema de archivos, etc.). La interfaz de usuario (UI) se renderiza en una WebView, pero está altamente optimizada y utiliza los componentes UI de Ionic que imitan la apariencia y el comportamiento nativo de cada plataforma.

💡 Filosofía Moderna de Ionic

La estrategia actual de Ionic se basa en dos pilares:

  • Componentes Web (Web Components): Desde Ionic 4, la biblioteca de UI de Ionic se distribuye como un conjunto de Componentes Web estándar. Esto significa que no depende de ningún framework en particular (como Angular, React o Vue), sino que puede ser usado con cualquiera de ellos (o incluso sin ninguno).
  • Capacitor: Es el sucesor moderno de Cordova. Capacitor es un "puente" nativo que conecta la WebView de tu aplicación con las APIs nativas del dispositivo. Es más moderno, flexible y fácil de mantener que Cordova.

Historia y Evolución (Versiones)

Ionic 1 (2013) - El inicio con AngularJS

La primera versión de Ionic estaba fuertemente acoplada a AngularJS 1.x. Revolucionó el desarrollo híbrido al proporcionar un conjunto de componentes UI robustos y directivas de Angular que facilitaban la creación de aplicaciones con apariencia nativa. Usaba Cordova como puente nativo.

Ionic 2 (2016) - Reescribiendo para Angular 2+

Con la llegada de Angular 2 (ahora solo "Angular"), Ionic 2 fue una reescritura completa. Adoptó TypeScript y la arquitectura basada en componentes de Angular. Esto mejoró drásticamente el rendimiento, la mantenibilidad del código y la experiencia del desarrollador.

Ionic 3 (2017) - Refinamiento y Lazy Loading

Fue una evolución de Ionic 2, alineándose con Angular 4+. Su principal mejora fue la introducción del "Lazy Loading" (carga diferida) a nivel de página, lo que mejoró significativamente los tiempos de arranque de la aplicación al cargar solo el código necesario para la vista actual.

Ionic 4 (2019) - "Ionic for Everyone" y Web Components

Esta fue la actualización más transformadora. El equipo de Ionic reescribió todos sus componentes UI como Web Components. Esto desacopló a Ionic de Angular, permitiendo que se usara oficialmente con React, Vue o cualquier otro framework. Introdujo Capacitor como la solución preferida sobre Cordova. El lema fue "Ionic para todos".

Ionic 5 (2020) - Diseño y Personalización

Se centró en refinar la experiencia de la interfaz de usuario. Introdujo un rediseño para asemejarse a la apariencia nativa de iOS 13, animaciones mejoradas, y una API de personalización de componentes mucho más potente a través de variables CSS, permitiendo una tematización más fácil.

Ionic 6 (2021) - Componentes Modernos y Vite

Mejoró componentes clave como ion-datetime y ion-modal. Introdujo el "Component Playground" para probar componentes de forma aislada. También comenzó a adoptar herramientas de compilación más rápidas como Vite en sus plantillas de inicio.

Ionic 7 (2023) - Simplificación y Rendimiento

La versión actual (usada en esta guía) se enfoca en simplificar la API. Hizo de Vite la opción por defecto para los starters de Angular, mejoró el control de superposiciones (Overlays como modals y popovers), y optimizó el rendimiento general, especialmente en la carga de la aplicación.

Estructura del Proyecto

Anatomía de un Componente (Página/Tab)

En un proyecto de Ionic con Angular, cada página o componente se compone típicamente de tres archivos principales que trabajan juntos:

  • *.page.html (El QUÉ)

    Es la plantilla (template). Define la estructura visual de la página usando etiquetas HTML y componentes de Ionic (como <ion-header>, <ion-content>, <ion-button>). Aquí es donde se realiza el "data binding" para mostrar datos (ej. {{ photo.title }}) y para escuchar eventos (ej. (click)="capturePhoto()").

  • *.page.ts (El CÓMO)

    Es la lógica (logic). Es un archivo TypeScript que contiene la clase del componente. Aquí se definen las variables (ej. photos: UserPhoto[]), se importan servicios (como el PhotoService) y se escriben los métodos (ej. capturePhoto()) que se ejecutan cuando el usuario interactúa con la vista.

  • *.page.scss (El ASPECTO)

    Son los estilos (styles). Es un archivo SCSS (Sass, un superconjunto de CSS) que define la apariencia visual específica de esa página. Los estilos definidos aquí están "encapsulados" o "scoped", lo que significa que solo se aplican a esta página y no afectan al resto de la aplicación.

El archivo *.page.spec.ts es para pruebas unitarias y no se utiliza en la aplicación en producción.

Estructura Detallada de Carpetas

Un proyecto de Ionic tiene una estructura de carpetas estándar. Las carpetas más importantes con las que interactuarás están dentro de src/:

  • src/app/: Es el corazón de tu aplicación. Contiene todos tus componentes, páginas, servicios y módulos.
    • src/app/tabs/: En esta plantilla, esta carpeta gestiona la navegación principal por pestañas.
    • src/app/tab1/, tab2/, etc.: Cada una es una página individual que se muestra en una pestaña.
  • src/assets/: Para archivos estáticos como imágenes, iconos, fuentes personalizadas o archivos JSON que no cambian.
  • src/theme/: Contiene variables.scss, donde defines los colores globales de tu aplicación (primario, secundario, etc.) y otros estilos globales.
  • src/environments/: Contiene archivos de configuración para diferentes entornos (ej. environment.ts para desarrollo y environment.prod.ts para producción). Aquí es donde pondrías claves de API, URLs de backend, etc.

@Component({
  selector: 'app-tab1',
  templateUrl: 'tab1.page.html',
  styleUrls: ['tab1.page.scss']

  constructor(
    private photoService: PhotoService,
    private alertController: AlertController,
    private actionSheetController: ActionSheetController

  ngOnInit() {
    this.loadPhotos();

    this.photos = await this.photoService.loadSaved();

  async capturePhoto() {
    // Capturar foto usando la cámara
    const photo = await Camera.getPhoto({

      source: CameraSource.Camera,
      quality: 90

    // Mostrar diálogo para agregar etiquetas
    const alert = await this.alertController.create({
      header: 'Etiquetar Foto',
      inputs: [
        {
          name: 'title',
          type: 'text',
          placeholder: 'Título de la foto'
        },
        {
          name: 'tags',
          type: 'text',
          placeholder: 'Etiquetas (separadas por comas)'
        }
      ],
      buttons: [
        {
          text: 'Cancelar',
          role: 'cancel'
        },
        {
          text: 'Guardar',
          handler: async (data) => {
            const tags = data.tags ? data.tags.split(',').map((t: string) => t.trim()) : [];

📄 src/app/services/photo.service.ts Servicio de Gestión de Fotos 
import { Injectable } from '@angular/core';
import { Camera, Photo } from '@capacitor/camera';
import { Filesystem, Directory } from '@capacitor/filesystem';
import { Preferences } from '@capacitor/preferences';
import { Platform } from '@ionic/angular';
import { Capacitor } from '@capacitor/core';

export interface UserPhoto {
  filepath: string;
  webviewPath?: string;
  title: string;
  tags: string[];
  date: Date;
}

@Injectable({
  providedIn: 'root'
})
export class PhotoService {
  public photos: UserPhoto[] = [];
  private PHOTO_STORAGE = 'photos';
  private platform: Platform;

  constructor(platform: Platform) {
    this.platform = platform;
  }

  public async addNewToGallery(photo: Photo, title: string, tags: string[]) {
    // Guardar la foto en el sistema de archivos
    const savedImageFile = await this.savePicture(photo);

    // Agregar la nueva foto al inicio del array
    const newPhoto: UserPhoto = {
      filepath: savedImageFile.filepath,
      webviewPath: savedImageFile.webviewPath,
      title,
      tags,
      date: new Date()
    };

    this.photos.unshift(newPhoto);

    // Guardar en Preferences
    await Preferences.set({
      key: this.PHOTO_STORAGE,
      value: JSON.stringify(this.photos)
    });
  }

  private async savePicture(photo: Photo) {
    // Convertir foto a base64
    const base64Data = await this.readAsBase64(photo);

    // Escribir el archivo en el directorio de datos
    const fileName = new Date().getTime() + '.jpeg';
    const savedFile = await Filesystem.writeFile({
      path: fileName,
      data: base64Data,
      directory: Directory.Data
    });

    if (this.platform.is('hybrid')) {
      // Mostrar la nueva imagen reescribiendo el path 'file://'
      return {
        filepath: savedFile.uri,
        webviewPath: Capacitor.convertFileSrc(savedFile.uri)
      };
    } else {
      // Usar webPath para mostrar la nueva imagen en la web
      return {
        filepath: fileName,
        webviewPath: photo.webPath
      };
    }
  }

  private async readAsBase64(photo: Photo) {
    // "hybrid" detectará Cordova o Capacitor
    if (this.platform.is('hybrid')) {
      // Leer el archivo como base64
      const file = await Filesystem.readFile({
        path: photo.path!
      });

      return file.data;
    } else {
      // Obtener la foto, leerla como blob y convertirla a base64
      const response = await fetch(photo.webPath!);
      const blob = await response.blob();

      return await this.convertBlobToBase64(blob) as string;
    }
  }

  private convertBlobToBase64 = (blob: Blob) => new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onerror = reject;
    reader.onload = () => {
      resolve(reader.result);
    };
    reader.readAsDataURL(blob);
  });

  public async loadSaved() {
    // Recuperar datos del array de fotos almacenado
    const { value } = await Preferences.get({ key: this.PHOTO_STORAGE });
    this.photos = (value ? JSON.parse(value) : []) as UserPhoto[];

    // Convertir las fechas de string a Date
    this.photos = this.photos.map(photo => ({
      ...photo,
      date: new Date(photo.date)
    }));

    // Mostrar las fotos leyendo el formato base64
    if (!this.platform.is('hybrid')) {
      for (let photo of this.photos) {
        // Leer los datos de cada foto guardada del sistema de archivos
        const readFile = await Filesystem.readFile({
          path: photo.filepath,
          directory: Directory.Data
        });

        // Solo plataforma web: cargar la foto como base64
        photo.webviewPath = `data:image/jpeg;base64,${readFile.data}`;
      }
    }

    return this.photos;
  }

  public async deletePhoto(photo: UserPhoto) {
    // Eliminar la foto del array
    this.photos = this.photos.filter(p => p.filepath !== photo.filepath);

    // Actualizar el array de fotos almacenado
    await Preferences.set({
      key: this.PHOTO_STORAGE,
      value: JSON.stringify(this.photos)
    });

    // Eliminar el archivo del sistema de archivos
    const filename = photo.filepath.substr(photo.filepath.lastIndexOf('/') + 1);
    await Filesystem.deleteFile({
      path: filename,
      directory: Directory.Data
    });
  }
}

Pruebas y Compilación

🌐 Prueba en el Navegador (Desarrollo)

Este es el modo que has estado usando. Inicia un servidor local con recarga en vivo.

# Iniciar servidor de desarrollo $ ionic serve ✔ Browser application bundle generation complete. ➜ Local: http://localhost:8100/
💡 Herramientas de Desarrollo
  • Presiona F12 para abrir DevTools
  • Activa el modo dispositivo (ícono de teléfono) para simular móvil
  • Los cambios se recargan automáticamente (Hot Reload)

🖥️ Compilación Web (Producción)

Cuando tu aplicación está lista para ser desplegada en un servidor web (como un sitio web estático), debes crear una compilación de producción optimizada.

# Crear compilación optimizada para producción $ ionic build # Esto ejecuta 'ng build --prod' bajo el capó ✔ Browser application bundle generation complete. ... (archivos minificados y optimizados)

Esto genera una carpeta www/ (o dist/ dependiendo de tu angular.json) con todos los archivos HTML, CSS y JS listos para subir a un hosting.

📱 Compilación PWA (Progressive Web App)

Una PWA es una aplicación web que se puede "instalar" en el dispositivo del usuario y funcionar sin conexión. Ionic con Angular tiene un excelente soporte para esto.

  1. Agregar Capacidades PWA

    Si no lo hiciste al inicio, puedes agregarlo con el CLI de Angular:

    $ ng add @angular/pwa ✔ Service worker configuration added. ✔ manifest.webmanifest and icons added.

    Esto añade un Service Worker (ngsw-worker.js) y un Web Manifest (manifest.webmanifest), que son los cerebros de la PWA.

  2. Construir la PWA

    El comando es el mismo que el de producción. El CLI de Angular detectará la configuración del Service Worker y la incluirá.

    $ ionic build

    Al desplegar el contenido de la carpeta www/ en un servidor (con HTTPS), los navegadores modernos permitirán a los usuarios "Instalar" la aplicación.

📱 Prueba en Dispositivo Android

  1. Construir el Proyecto Web

    Crea la versión web optimizada que será empaquetada.

    $ ionic build
  2. Agregar Plataforma Android

    Este comando solo se ejecuta una vez para crear la carpeta android/.

    $ ionic cap add android
  3. Sincronizar Cambios

    Este es el comando clave. Copia los archivos web de www/ al proyecto nativo de Android y actualiza los plugins.

    $ ionic cap sync android
  4. Abrir en Android Studio

    Abre el proyecto nativo en el IDE de Android.

    $ ionic cap open android
  5. Ejecutar en Dispositivo

    Desde Android Studio: Conecta tu dispositivo (con depuración USB activada) y presiona "Run" (▶️).

⚠️ Permisos de Android

Capacitor gestiona muchos permisos automáticamente. Para la cámara, se añadirán al sincronizar. Asegúrate de que tu android/app/src/main/AndroidManifest.xml los tenga.

<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

🍏 Prueba en Dispositivo iOS

🚫 Requisito Indispensable

Para compilar aplicaciones de iOS, necesitas tener una computadora con macOS y Xcode instalado (disponible gratis en la App Store de Mac).

  1. Construir el Proyecto Web
    $ ionic build
  2. Agregar Plataforma iOS

    Crea la carpeta ios/ con el proyecto de Xcode.

    $ ionic cap add ios
  3. Sincronizar Cambios

    Copia los archivos web de www/ al proyecto de iOS.

    $ ionic cap sync ios
  4. Instalar Dependencias Nativas (CocoaPods)

    Capacitor usa CocoaPods para gestionar plugins en iOS.

    # Navegar a la carpeta nativa $ cd ios/App # Instalar los pods (plugins) $ pod install # Regresar a la raíz $ cd ../../
  5. Abrir en Xcode

    Abre el proyecto nativo en Xcode.

    $ ionic cap open ios
  6. Ejecutar en Dispositivo

    Desde Xcode: Conecta tu iPhone o iPad. Deberás seleccionar tu cuenta de desarrollador (puede ser una gratuita) en la pestaña "Signing & Capabilities" y luego presionar "Run" (▶️).

🔄 Flujo de Desarrollo Continuo

Tu flujo de trabajo diario será:

  1. Desarrollar y probar en el navegador: ionic serve.
  2. Cuando estés listo para probar en un dispositivo: ionic build.
  3. Luego: ionic cap sync android (o ios).
  4. Finalmente: ionic cap open android (o ios) para ejecutar desde el IDE nativo.
